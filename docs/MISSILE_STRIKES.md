# Ракетные удары командира

Командир фракции выбирает цель на полноэкранной карте: ПКМ → «Ракетный удар» → профиль
ракеты → подтверждение. Сервер повторно проверяет роль, команду, координаты, склад, кулдаун,
активную ракету и защищённые зоны; данным клиента он не доверяет.

## Требования

- SuperbWarfare должен быть установлен на сервере и клиентах. BaseMod по-прежнему может
  загружаться без него, но запуск ракет будет запрещён.
- Команда должна быть привязана к складу командой
  `/pjm capturepoint warehouse <фракция> <склад>`.
- На складе списываются очки пула `SUPPLY`. OP уровня 2 не оплачивает пуск и игнорирует
  кулдаун, но должен состоять в scoreboard-команде.
- Рядом с целью должен быть игрок, иначе обычный командир не может заказать удар в
  незагруженную пустоту. OP может обойти эту проверку.

## Конфигурация

При первом запуске создаётся `config/pjmbasemod/missiles/missiles.json` с профилями:

- `iskander_k`
- `storm_shadow`
- `kh_101`
- `iskander_m`
- `flamingo`

Перезагрузка: `/pjm config reload missiles` или `/pjm config reload all`.

Ключевые поля профиля:

- `trajectory`: `cruise` или `ballistic`;
- `supplyCost`, `cooldownSeconds`, `flightSeconds`;
- `spawnDistance`, `cruiseHeight`, `terminalDiveDistance`, `ballisticApex`;
- `weaveAmplitude`, `weaveCycles` — змейка захода; терминальный заход — плавный перелом курса вниз;
- `damage`, `radius`, `hitPoints`, `shotDownPower`;
- `destroyBlocks`: по умолчанию `false` для сохранности карты;
- `teams`: список id scoreboard-команд, которым доступна ракета (пустой — всем);
- кулдаун (`cooldownSeconds`) считается отдельно на каждую пару команда+ракета.

## Оповещения

При пуске своя команда мгновенно получает уведомление с названием ракеты и янтарную
зону поражения на карте. Все остальные — через ~8.5 секунды: красную зону на карте
(без названия) и, если находятся ближе 256 блоков к цели, эвакуационное уведомление.
Зона предупреждения висит 25 секунд; после детонации на карте остаётся отметка
поражения (45 секунд, с плавным затуханием).

`radius` передаётся непосредственно в `SBW CustomExplosion`. У текущего форка урон может
доставать примерно до `2 × radius`, поэтому запрет базовых зон использует расширенный радиус.
Профиль активной ракеты копируется в сущность: reload посреди полёта не меняет уже заказанный
удар.

## Модели

Одна сущность выбирает ресурсы по id ракеты:

```text
assets/pjmbasemod/geo/entity/missiles/<id>.geo.json
assets/pjmbasemod/textures/entity/missiles/<id>.png
```

До добавления конкретного ресурса используется модель Х-101. Модель и текстура Х-101 уже
лежат под id `kh_101`.

## Звук

Для полёта используются mono-версии `missile_engine.ogg` и `rocket_engine.ogg`. Из длинного
звука двигателя подготовлены три фильтрованных loop-слоя: `far`, `medium`, `close`. Клиент
привязывает все слои к движущейся сущности и плавно смешивает их по расстоянию. Дополнительно:
per-missile окклюзия (нет прямой видимости до ракеты — все слои приглушаются до ~30% со
сглаживанием) и одноразовый woosh `missile.flyby` (6 рандомных вариантов из Squad) в момент
максимального сближения ближе 60 блоков. Звук ведётся по виртуальным трекам из
`MissileAudioSyncPacket` (сервер шлёт позицию раз в 2 тика игрокам в радиусе 750 блоков),
поэтому ракету слышно независимо от прогруза сущности; far-слой достаёт до ~700 блоков.
Взрыв, частицы, тряску камеры и его дистанционные звуки создаёт SuperbWarfare. Ракета
спавнится за `spawnDistance` (по умолчанию 2500) блоков от цели и тянет реактивный след
из частиц (force-рассылка игрокам в радиусе 400 блоков).

Перед публичным распространением сборки следует отдельно проверить лицензию исходного
набора JSRS 2025 на право включать аудиофайлы в модпак или JAR.
