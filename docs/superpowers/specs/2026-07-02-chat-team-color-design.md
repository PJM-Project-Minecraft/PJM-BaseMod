# Окраска чат-сообщений цветом scoreboard-команды

**Дата:** 2026-07-02
**Статус:** одобрено

## Цель

Ник и текст сообщения игрока в чате красятся цветом его scoreboard-команды:

```
[G] ⟦иконка⟧[SGT] Ник: привет всем
                  └──── всё цветом команды ────┘
```

Тег режима (`[G]`/`[L]`) и бейдж ранга сохраняют свои цвета.

## Решение (вариант A)

Меняется только `common/chat/ChatService` (src/main). Сеть, `PjmNetworking.VERSION`,
клиент не трогаются — сообщение приходит клиенту готовым `Component` через
`displayClientMessage`.

### Хелпер `teamColorRgb(ServerPlayer sender)` → `@Nullable Integer`

Приоритет источников цвета:

1. `sender.getTeam()` отсутствует → `null` (поведение как сейчас — цвет режима).
2. У scoreboard-команды задан цвет (`team.getColor().getColor() != null`) →
   его RGB. Работает для любой команды, включая созданные вручную `/team`.
3. Цвет не задан (RESET), но команда есть в `Config.getTeams()` →
   палитра `FrontlineTeams.color()` (фолбэк по индексу команды).
4. Цвет не задан и команда не сконфигурирована → `null`.

### `decorate()`

Если `teamColorRgb != null` — компонент ника `"Ник: "` и копия сообщения
(`raw.copy()`) получают `withColor(rgb)`. Стиль ставится на корень: явные цвета
внутри сообщения (если есть) не перезатираются. Все режимы чата (GLOBAL /
LOCAL / TEAM) — одинаково, всё идёт через общий `decorate`.

## Что НЕ делаем

- Флаг конфига для отключения — YAGNI (вариант C отклонён).
- Ограничение только сконфигурированными командами (вариант B отклонён).

## Проверка

- `./gradlew compileJava` + `./gradlew compileClientJava`.
- Визуально в игре (из пути без `!`) — за пользователем: игрок в команде с
  цветом, в команде без цвета, без команды.
