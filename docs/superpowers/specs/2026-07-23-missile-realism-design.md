# Реализм стратегических ракет: баллистика, фазы крылатых, трек на карте

Дата: 2026-07-23. Статус: утверждено пользователем.

## Цель

Три независимых улучшения подсистемы ракетных ударов (`common/missile/`, `common/entity/StrategicMissileEntity`):

1. Реалистичная квазибаллистическая траектория (вертикальный старт → парабола → терминальный доворот).
2. Реалистичные фазы крылатой ракеты (низкий марш → pop-up у цели → крутое пикирование).
3. Живой трек ракеты на полноэкранной карте — стрелка, только для команды, запустившей ракету.

Архитектурный инвариант сохраняется: позиция ракеты — чистая функция от `elapsedTicks`
(детерминизм, устойчивость к фризам чанков и save/load), никакой per-tick физики со состоянием.

## 1. Баллистика (вариант A — параметрическая кусочная кривая)

`BallisticTrajectory` перестраивается на три C1-непрерывных участка:

- **Буст** — квадратичная Безье: старт строго вверх (горизонтальная скорость ≈ 0 при t=0),
  плавный завал на курс; конечная касательная совпадает с начальным наклоном параболы.
  Занимает ~5–8% горизонтальной дистанции; горизонтальная скорость нарастает ease-in
  (тайм-варп по доле горизонтали), общее время полёта остаётся `flightSeconds`.
- **Марш** — честная парабола `y = хорда + 4·apex·v(1−v)`, апогей из `ballisticApex` (JSON как есть).
- **Терминал** — если наклон параболы у цели положе `IMPACT_ANGLE_DEGREES` (80°),
  работает существующий плавный доворот до 80° (`IMPACT_SLOPE`-механизм);
  если парабола сама круче — доворот не включается.

### API

`altitudeAt(t, …)` → `sample(t, startY, targetY, horizontalDistance, apexHeight)`,
возвращает `(horizontalFraction, altitude)`. `StrategicMissileEntity.ballisticPosition`
лерпит XZ по `horizontalFraction`, а не по `t`. Поворот модели уже считается из вектора
движения — вертикальный старт и доворот отображаются без доработок.

Короткие маршруты: текущий fallback-профиль сохраняет непрерывность и угол входа.

## 2. Фазы крылатой ракеты

Профиль ПКР (Калибр/Томагавк) в `cruisePosition`:

1. Разгонный набор до крейсерской — есть (сглаживание 2-го порядка).
2. Марш низко с огибанием рельефа — есть (`cruiseHeight` из JSON; для реализма в профилях ставить ~12–18).
3. **Pop-up**: за `popupDistance` блоков до цели — горка до `popupHeight` над целью.
4. Пикирование из вершины горки существующим `dive²`-механизмом — угол входа резкий.

### Конфиг

`MissileDefinition`: новые опциональные JSON-поля `popupDistance` (default 120)
и `popupHeight` (default 60). `popupDistance: 0` — старое поведение без горки.
Сеть и клиент не затрагиваются.

## 3. Трек ракеты на карте (только своей команде)

- Новый S→C пакет `MissileTrackPacket(uuid, dimension, x, z, yaw, active)`.
- Сервер шлёт из `StrategicMissileEntity.tick()` раз в 10 тиков членам команды пуска
  через `sendToTeam` без лимита дистанции; при детонации/удалении — финальный `active=false`.
- Приватность гарантирует сервер: чужим командам пакет не уходит; у них остаётся
  существующая отложенная зона предупреждения.
- Клиент: `ClientMissileState` — мапа uuid → трек, TTL ~3 с без апдейтов (стрелка гаснет сама).
- `MapOverlays`: стрелка (треугольник), повёрнутая по yaw, в позиции ракеты; для обоих типов ракет.
- По шаблону подсистем: регистрация в `PjmNetworking` (бамп `VERSION` "12" → "13"),
  метод в `ClientPacketProxy` (default noop) + реализация в `ClientPacketHandlersImpl`.

## Тесты

`BallisticTrajectoryTest` переписывается: вертикальность старта (горизонтальная
компонента ≈ 0, вертикальная > 0 при t≈0), C1-непрерывность на стыках (без скачков),
достижение апогея, угол входа 80° на длинном маршруте, корректность короткого маршрута.
Фазы крылатых и сеть — верификация `compileJava` + `compileClientJava` + внутриигровая
проверка пользователем (runClient из этого каталога не работает — `!` в пути).

## Вне скоупа

Per-tick физика, скоростной профиль баллистики (разгон/торможение вне буста), визуал
фаз (отсечка двигателя), изменения для вражеской команды, локализация новых ключей
не требуется (стрелка без текста).
